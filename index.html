Real Issues I Solved (IT Support & QA)

DOMMatrix Deployment Error

Type: Production Bug - Category: Environment Mismatch - Severity: High

Problem

App worked perfectly on localhost. After deploying to Vercel, PDF parsing completely broke with error: "DOMMatrix is not defined"

Impact: Resume analysis feature was completely unavailable in production, affecting all users.

Investigation(2 days)

  • Cleared cache and redeployed — no change
  • Checked environment variables — correct
  • Updated package versions to rule out dependency version issues — no change
  • Verified local vs production behavior to confirm issue was environment-specific
  • Shifted focus from code to runtime behavior — this was the key turning point
  • Read Node.js runtime docs to understand browser vs server API availability

Root Cause

The issue was caused by an environment mismatch. The library used (pdfjs-dist) depends on browser-specific APIs (DOMMatrix), which are not available in the Node.js runtime used in production. Same code, different runtime — causing failure only after deployment.

Solution

    Replaced the incompatible library (pdfjs-dist) with a Node.js-compatible alternative (pdf-parse) and redeployed successfully.

    Once the root cause was identified, the fix took less than 20 minutes — the investigation was the hard part, not the fix.

Key Lessons:

"It works on my machine" is not a diagnosis. The right question is: what environment is this actually running in?

This mirrors the core of IT Support work — environment mismatches, permission conflicts, and configuration drift are among the most common real-world issues. Systematic root cause analysis, not guessing, is always the path to resolution.
