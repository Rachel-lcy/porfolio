Troubleshooting Case Study

DOMMatrix Deployment Error

Problem

App worked perfectly on localhost. After deploying to Vercel, PDF parsing completely broke with error: "DOMMatrix is not defined"

Investigation(2 days)

  • Cleared cache and redeployed — no change
  • Checked environment variables — correct
  • Updated package versions — no change
  • Stopped guessing, started reading the Node.js runtime docs

Root Cause

pdfjs-dist relies on DOMMatrix — a browser-only API. Localhost runs in the browser. Vercel runs server-side in Node.js. Node.js has no concept of DOMMatrix. Same code. Two completely different environments.

Solution

    Replaced pdfjs-dist → pdf-parse (a library built specifically for Node.js server environments) Redeployed. Fixed in 20 minutes once root cause was understood.

Key Lessons:

"It works on my machine" is not a diagnosis. The right question is: what environment is this actually running in?

→ This mirrors exactly what IT Support deals with daily: environment mismatches, permission issues, and systematic root cause analysis.
